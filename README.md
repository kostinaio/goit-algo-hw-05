# Task3 Висновки щодо швидкостей алгоритмів пошуку підрядка

## Вступ
У цьому документі представлено результати вимірювань швидкості трьох алгоритмів пошуку підрядка: Боєра-Мура, Кнута-Морріса-Пратта (КМП) та Рабіна-Карпа. Вимірювання проводились для двох текстів, кожен з яких мав два види підрядків: існуючий в тексті та неіснуючий. Результати надані у вигляді часу виконання алгоритмів (в секундах) для 1000 запусків кожного алгоритму.

## Результати вимірювань

### Текст 1
- **Boyer-Moore**
  - Існуючий підрядок: 0.2534 сек
  - Неіснуючий підрядок: 0.3891 сек
- **KMP**
  - Існуючий підрядок: 1.1685 сек
  - Неіснуючий підрядок: 4.2748 сек
- **Rabin-Karp**
  - Існуючий підрядок: 2.2386 сек
  - Неіснуючий підрядок: 6.7277 сек

### Текст 2
- **Boyer-Moore**
  - Існуючий підрядок: 0.2409 сек
  - Неіснуючий підрядок: 0.3822 сек
- **KMP**
  - Існуючий підрядок: 2.5769 сек
  - Неіснуючий підрядок: 6.6432 сек
- **Rabin-Karp**
  - Існуючий підрядок: 4.1726 сек
  - Неіснуючий підрядок: 10.5866 сек

## Висновки

### Швидкість алгоритмів для тексту 1
Для тексту 1 алгоритм Боєра-Мура показав найшвидші результати як для існуючого підрядка, так і для неіснуючого підрядка. КМП і Рабіна-Карпа значно повільніші, причому алгоритм Рабіна-Карпа показав найгірші результати.

### Швидкість алгоритмів для тексту 2
Для тексту 2 аналогічно алгоритм Боєра-Мура виявився найшвидшим як для існуючого, так і для неіснуючого підрядка. Алгоритм КМП показав середню швидкість, тоді як алгоритм Рабіна-Карпа знову виявився найповільнішим.

### Загальний висновок
Загалом, алгоритм Боєра-Мура виявився найефективнішим серед трьох алгоритмів для пошуку підрядка у обох текстах. Це пояснюється його оптимізаціями для сканування тексту справа наліво і використанням таблиць зсувів для швидкого пропуску непотрібних порівнянь. Алгоритми КМП та Рабіна-Карпа показали значно гірші результати, особливо для неіснуючих підрядків, де час їх виконання був найбільшим.

### Рекомендації
Для пошуку підрядків у текстах рекомендується використовувати алгоритм Боєра-Мура, особливо у випадках, коли очікується велика кількість неіснуючих підрядків або коли необхідна висока швидкість обробки великих текстів.
